# Windows 交互语音助手 - 功能规划大纲

## 1. 目标与愿景

- 目标：构建"即说即用"的 Windows 语音超级助手，支持短期（基础控制）与长期（扩展插件）两条路径。
- 用户群：电脑白领、开发人员、学习者、残障用户。
- 核心价值：
  - 语音执行常用系统任务
  - 自定义指令、可扩展插件
  - 低门槛部署、可无人值守

## 2. 产品结构与版本路线

### V1（MVP）- 已完成
- 唤醒词（可配置）
- 本地语音识别 + 纯文本命令解析
- 关键命令：应用控制、系统设置、文件操作、时间管理
- 反馈：TTS + Windows 通知
- 日志与异常处理

### V2（增强）- 已完成 ✅
- **NLU 意图+槽位** — FuzzyRegex 增强正则 + 预备 HybridNLUEngine（sentence-transformers）
- **语音命令纠错** — 基于拼音+Levenshtein 的 ASR 识别纠错（PhoneticCorrector）
- **抖音键盘控制** — pyautogui 跨应用快捷键自动化（播放/暂停/点赞/评论等）
- `intents.json` 配置式扩展
- 插件体系

### V3（智能）- 规划中
- 线上语言模型扩展
- 自学习推荐
- 场景模式（会议/演示）
- UI 仪表盘
- 跨设备联动

## 3. 核心模块划分

1. 命令入口（main.py）
2. 解析层（intents.py / nlu/）
3. 执行层（executor.py）
4. 反馈层（feedback.py）
5. 配置层（config.py / config.yaml）
6. 插件层（plugins.py）

## 4. 已实现功能清单

### 基础命令（V1）
- 系统控制（高）：打开/关闭应用、音量设置、定时提醒
- 文件操作（中）：保存文本到文件夹
- 应用管理（高）：扫描本地程序、智能匹配、中英文别名
- 时间与通知（高）：定时闹钟、TTS 播报、Windows 通知

### 增强功能（V2）
- **ASR 纠错**：修正"打一微信"→"打开微信"等语音识别错误
- **FuzzyRegex 增强匹配**：模糊应用名提取、关键词容错
- **抖音键盘控制**：space/l/i/f/s 等快捷键触发，支持播放、点赞、评论、滚动等操作
- **HybridNLU 预备**：sentence-transformers 语义匹配（需安装额外依赖）

## 5. 技术实现

| 模块 | 技术方案 |
|------|---------|
| 语音引擎 | sounddevice + Google STT（recognize.py）|
| NLU | Regex → FuzzyRegex → Hybrid Engine |
| ASR 纠错 | pypinyin + rapidfuzz |
| 键盘自动化 | pyautogui |
| TTS | pyttsx3 / win10toast |
| 配置 | YAML |
| 插件 | importlib 动态加载 |
